# 梆梆加固脱壳指南 — E听说中学

## 一、脱壳方案对比

| 方案 | 难度 | Root需求 | 适用壳类型 | 推荐度 |
|------|------|----------|-----------|--------|
| BlackDex | ⭐ | ❌ 不需要 | 第一代(整体加密) | ⭐⭐⭐⭐⭐ |
| frida-dexdump | ⭐⭐⭐ | ✅ 需要 | 第一/二代 | ⭐⭐⭐⭐ |
| 反射大师 | ⭐⭐ | ✅ 需要+Xposed | 第一/二代 | ⭐⭐⭐ |
| FART | ⭐⭐⭐⭐ | ✅ 需要刷机 | 第二代(代码抽取) | ⭐⭐⭐ |
| AI辅助(rev-dex-dumper) | ⭐ | ✅ 需要 | 通用 | ⭐⭐⭐⭐ |

> E听说中学使用的是**梆梆免费版**(非企业版)，属于第一代整体DEX加密，
> BlackDex即可脱壳，无需Root。

---

## 二、BlackDex 脱壳步骤（推荐，免Root）

### 1. 下载安装 BlackDex
- GitHub: https://github.com/CodingGay/BlackDex
- 选择对应架构: BlackDex32(32位) / BlackDex64(64位) / BlackDex-all(通用)

### 2. 执行脱壳
1. 打开 BlackDex
2. 在应用列表中找到 **E听说中学** (com.ets100.secondary)
3. 点击脱壳按钮，等待完成
4. 脱壳文件保存在:
   ```
   /storage/emulated/0/Android/data/top.niunaijun.blackdexa32/com.ets100.secondary/
   ```

### 3. DEX修复与重命名
- 脱出的dex文件需要修复（BlackDex脱出的可能头部损坏）
- 使用 MT管理器 或 jadx 直接打开查看
- 重命名为 `classes.dex`, `classes2.dex`, ... 顺序编号

## 三、frida-dexdump 脱壳步骤（需Root）

### 1. 环境搭建
```bash
# PC端安装frida工具链
pip install frida-tools frida-dexdump

# 下载frida-server到手机
# 从 https://github.com/frida/frida/releases 下载对应架构版本
adb push frida-server-16.x.x-android-arm64 /data/local/tmp/fs
adb shell "chmod 755 /data/local/tmp/fs"
```

### 2. 启动frida-server
```bash
adb shell "su -c 'killall frida-server'"  # 先杀旧进程
adb shell "su -c '/data/local/tmp/fs &'"   # 后台启动
```

### 3. 执行脱壳
```bash
# 先在手机上打开E听说中学

# 确认进程存在
frida-ps -Ua | grep ets100

# 方式一: 按包名脱壳
frida-dexdump -U -f com.ets100.secondary

# 方式二: 按进程名脱壳
frida-dexdump -U -n E听说中学
```

### 4. 梆梆Anti-Frida绕过（如遇到闪退）
梆梆企业版可能检测Frida，需要魔改frida-server:
- 修改frida特征字符串
- 使用 `hluda` 魔改版frida-server
- 或使用以下脚本绕过:
```javascript
// 绕过maps文件扫描
Java.perform(function() {
    var File = Java.use("java.io.File");
    File.exists.implementation = function() {
        var path = this.getAbsolutePath();
        if (path.indexOf("frida") !== -1 || path.indexOf("re.frida") !== -1) {
            return false;
        }
        return this.exists();
    };
});
```

## 四、脱壳后分析流程

### 1. 反编译
```bash
# 使用jadx打开所有脱壳dex
jadx-gui classes.dex classes2.dex classes3.dex ...
```

### 2. 关键类搜索
在jadx中搜索以下关键词定位VIP/E卡逻辑:
- `isVip` / `isExpired` / `isEcard`
- `vipStatus` / `vipType` / `ecardStatus`
- `expireTime` / `validDate` / `deadline`
- `BuyEcardWebView` (已知购买入口)
- `ic_vip_lock_flag` (已知VIP锁图标引用)

### 3. 定位修改点
找到以下类型的方法后记录完整类名和方法签名:
- 返回boolean的VIP判断方法 → 修改返回值为true
- 返回long/int的过期时间方法 → 修改返回值为极大时间戳
- 返回String的VIP类型方法 → 修改返回值为最高等级

## 五、脱壳后的替代方案

如果脱壳成功，除了Xposed Hook，还可以直接修改smali:

### 使用apktool反编译修改
```bash
# 反编译
apktool d test.apk -o test_dir

# 修改smali代码 (在test_dir/smali*中找到目标方法)
# 将 isVip() 返回值改为 const/4 v0, 0x1 → return v0

# 重新打包
apktool b test_dir -o test_patched.apk

# 签名
apksigner sign --ks mykey.jks --out test_signed.apk test_patched.apk
```

> 注意: 修改smali后需要重新签名，签名变更可能导致:
> - 微信/支付宝支付不可用
> - 登录态丢失
> - 应用检测到签名不匹配闪退
>
> 因此**Xposed Hook方式更安全**，不修改原APK。
